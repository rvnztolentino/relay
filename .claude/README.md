# Claude Prompt Folder

This folder stores reusable prompts for agent-driven project work.

`CLAUDE.md` is the lightweight router. It should not cause Claude to read every Markdown file by default.

## Files

- `vision.md` - task source or implementation plan. It may be blank, generated by `agents/engineering-manager.md`, or already complete from assigned work.
- `commands/instructions.md` - main operating instructions for task execution, branching, completion, and artifact outputs.
- `rules/branch-strategy.md` - branch naming, branch creation, and multi-branch progress tracking rules.
- `agents/engineering-manager.md` - creates or updates `vision.md` with an implementation plan and planned verification steps.
- `agents/documentation-coordinator.md` - creates pull request summaries and manual testing notes after implementation.
- `agents/systems-analyst.md` - creates or updates the canonical project reference from repository contents.
- `agents/release-engineer.md` - creates a concise pull request description and completed-work summary.
- `agents/code-reviewer.md` - reviews code changes for issues, risks, and missing tests.
- `agents/security-officer.md` - reviews code for security concerns.
- `agents/qa-lead.md` - checks completed work against `vision.md`.
- `agents/documentation-writer.md` - generates a project `README.md`.
- `agents/markdown-editor.md` - cleans and organizes notes into Markdown without changing information.
- `agents/work-log-writer.md` - creates short work logs.

## Standard Agent Outputs

- `vision.md` - implementation plan or task source
- `.claude/tests.md` - manual tests created after implementation is done
- `.claude/pull-request.md` - pull request description, completed-work summary, changelog, and risk notes

These paths are centralized in `commands/instructions.md`. Specialized prompts should only repeat the specific output path they directly write.

## Branch File

- `branch.md` - selected branch strategy for the current task and progress tracker when work is split across multiple branches, including branch status and PR status.

Branches are created locally only. The agent must not push changes.

For multi-branch work, the agent completes the current branch, updates `branch.md`, generates the branch artifacts, and stops for user review before continuing to the next branch.

When you tell the agent a completed branch has an active PR, it should mark that branch's PR status as `Active` in `branch.md`, then start only the next planned branch.
